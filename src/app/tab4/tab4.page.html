Privacy & Security Analysis (ULO 4)

1. RESTful API Security

This application communicates with a remote server via HTTPS, ensuring that all data transmitted between the mobile app and the server is encrypted. This prevents Man-in-the-Middle (MITM) attacks where attackers could intercept sensitive inventory data.

HTTPS encryption for all API calls URL encoding for item names in API paths Proper HTTP methods (GET, POST, PUT, DELETE) 2. Input Validation & XSS Prevention

Angular's built-in templating engine automatically sanitizes values bound to the DOM, effectively preventing Cross-Site Scripting (XSS) attacks. TypeScript interfaces enforce strict data types, ensuring numeric fields (Price and Quantity) only accept valid numbers.

TypeScript type checking for all data fields Required field validation before API calls Numeric validation for price and quantity Angular sanitization prevents XSS attacks 3. Data Privacy Considerations

The application handles sensitive inventory data including supplier information and pricing. Privacy measures ensure that this business data is only accessible to authorized users and is properly managed during transmission and storage.

No local storage of sensitive data Server-side auto-increment for IDs (prevents tampering) Protected items (e.g., "Laptop") cannot be deleted 4. Mobile-Specific Security

Mobile applications face unique security challenges including device theft, unsecured Wi-Fi networks, and platform-specific vulnerabilities. Ionic/Capacitor provides security features to address these concerns.

Cross-platform security with Ionic framework Error handling without exposing sensitive information User confirmation for destructive actions (delete) 5. Security Best Practices Implemented

Authentication & Authorization

Server validates all requests and enforces deletion protections on critical records.

Data Validation

Both client-side and server-side validation ensure data integrity.

Error Handling

Graceful error handling prevents application crashes and information leakage.

User Experience

Clear feedback and confirmation dialogs prevent accidental data loss.
